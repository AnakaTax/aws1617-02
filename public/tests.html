Tests

Click on the buttom to run tests in your Postman application.

Which tests do we run?

Here is the sequence!

  1. Get all researchers - OK
      2. This test tries to get all registered researchers in the system, having a successful result.
  2. Get a specific researcher - OK
      3. This test tries to get one registered researcher in the system only, having a successful result.
  3. Get a specific researcher - FAIL
      4. This test tries to get an existing researcher in the system, having a failed result because there is not any researcher with specified ID Card.
  4. Create a new researcher - OK
      5. This test tries to create a new researcher in the system, having a successful result.
  5. Create an existing researcher - FAIL
      6. This test tries to create a new researcher in the system, having a failed result because there is an existing researcher with the same ID Card.
  6. Create a wrong researcher - FAIL
      7. This test tries to create a new researcher in the system, having a failed result because the researcher on the request body is wrong.
  7. Update a specific researcher - OK
      8. This test tries to update an existing researcher in the system, having a successful result.
  8. Update a not-existing researcher - FAIL
      9. This test tries to update an existing researcher in the system, having a failed result because there is not an existing researcher with the same ID Card.
  9. Delete a specific researcher - OK
      10. This test tries to remove an existing researcher in the system, having a successful result.
  10. Delete a not-existing researcher - FAIL
      11. This test tries to remove an existing researcher in the system, having a failed result because there is not an existing researcher with the same ID Card.
